## LostFrameWork

#### 1.说明
很多开发过程中存在大量重复轮子，因此该架构作为个人整合框架进行开发。  
后面会加入编辑器功能，对这些模块进行一个开关设置，项目结束后再进行整理，框架优先度降低。  
对于开发而言，并不是功能越多越好，根据需求逐渐扩展。最小开发规模，因此对一些可视化工具暂时不做支持。  
可视化工具会增加项目开发和维护难度，不建议程序员使用可视化进行开发游戏，这个部分应当由策划和美术进行负重非代码部分的配置。
